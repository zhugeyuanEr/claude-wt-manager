# Worktree 规划 (旧版)

分析项目结构，自动计算推荐的 worktree 数量。

## 使用方式

```
/wt-planb                   # 自动分析，推荐数量
/wt-planb 3                 # 指定 3 个 worktree
```

## 分析步骤

1. 扫描项目结构 (frontend/backend/modules)
2. 分析代码规模 (行数、文件数)
3. 评估推荐数量

## 推荐数量

```
基础: 2 (前端 + 后端)
+1 if 文件数 > 100
+1 if 文件数 > 300
+1 if 文件数 > 500
+1 if 文件数 > 800
+1 if 存在 tests/ 目录
+1 if 存在 .github/ 目录
最大: 6
```

## 与 wt-plan 的区别

| 功能 | wt-plan | wt-planb |
|------|---------|----------|
| 项目分析 | ✓ | ✓ |
| 推荐数量 | ✓ | ✓ |
| 交互式输入工作内容 | ✓ | ✗ |
| 生成规划表 | ✓ | ✗ |
| 生成 git 命令 | ✓ | ✗ |

**提示**: 使用 `/wt-plan` 获取完整的规划生成功能。
